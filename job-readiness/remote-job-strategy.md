# Remote US AI Job Strategy

## Market Reality (2026)

- Majority of AI Engineer postings offer hybrid or remote
- Competition is **national** — portfolio quality matters more than location
- US employers often require: work authorization, 4+ hr timezone overlap

## Profile Optimization for Remote

1. **GitHub > geography** — invest in README metrics and demos
2. **Async communication proof** — ADRs, detailed PRs, Loom walkthroughs
3. **LinkedIn:** "Open to remote (US)" + specific stack keywords
4. **Timezone:** State availability explicitly (e.g., "ET 9am–6pm")

## Best Boards for Remote AI

| Board | Notes |
|-------|-------|
| LinkedIn | Filter remote + "AI Engineer" |
| Wellfound | Startup remote-heavy |
| Otta / Cord | Curated tech |
| RemoteOK | Volume; filter carefully |
| Company career pages | Databricks, Anthropic, OpenAI, Scale — check directly |

## Red Flags

- "Remote" but requires occasional unpaid relocation
- No engineering blog or eng hiring page
- Vague "AI startup" without product description
- Unlimited take-home (>4h unpaid labor)

## Interview Adjustments

- **Over-communicate** in system design — narrate diagram as you draw
- **Stable setup** — wired internet, backup hotspot for video
- **Whiteboard tools** — Excalidraw, Miro practiced in advance

## Compensation

- Remote tier-1 companies: often **90–100%** of SF base bands
- Geo-adjusted companies: verify band before late-stage interviews
- Negotiate on **total comp** — remote candidates sometimes lowballed on equity
